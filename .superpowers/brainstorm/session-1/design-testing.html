设计 A：测试策略

用最小成本验证关键路径

三层验证

1. 静态冒烟测试

  • HTML 文件能直接在浏览器打开
  • Azure SDK CDN 能正常加载
  • 页面在手机 Safari 上布局不崩

2. 模拟数据测试

  • 不调用 Azure，用写死的评估 JSON 跑通 UI
  • 验证高亮、分数、drill 切换是否正常
  • 验证对齐算法对缩写/标点的处理

3. 真实录音测试（iOS 重点）

  • 在 iPhone Safari 上请求麦克风权限
  • 录制 30 秒英文，确认 Azure 返回结果
  • 测试开始/停止/重录的稳定性
  • 测试 drill 模式下反复录音是否卡顿

由于你主要在 iOS 上使用，真实录音测试以 iPhone Safari 为最高优先级，桌面浏览器作为开发调试工具。
