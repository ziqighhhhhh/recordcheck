设计 A：数据流

从用户点击“开始”到看到高亮结果的流程

录音与评估流程
1 用户粘贴原文 + 填入 Azure Key/Region
2 浏览器请求麦克风权限 → 开始录音
3 音频流 → Azure Speech SDK（发音评估模式）
4 Azure 返回：识别文本 + 词/音素级分数
5 前端对齐：原文 vs 识别结果（忽略大小写、标点、缩写）
6 UI 高亮：绿/黄/红 + 总体与维度分数
7 用户点击问题区域 → 进入 Drill（句/词循环录音）
对齐算法要点
  • 归一化：I’ve → ive，I’m → im，去掉标点，统一小写
  • 顺序匹配：按词序逐词对比原文与识别文本
  • 漏词：原文有但识别无 → 标红
  • 错词：位置相同但文本不同 → 标红/黄（看 Azure 发音分）
  • 多词：识别有但原文无 → 标黄提示

关键依赖：Azure Speech JS SDK 从 CDN 加载；录音使用浏览器 MediaRecorder 或 SDK 内置麦克风。
