# 英语朗读练习工具设计文档

## 1. 目标

帮助用户通过反复朗读英文原文，快速定位自己读得不好的地方（漏读、错读、发音、流畅度、语调），并针对问题句/词循环练习。

## 2. 关键决策

| 问题 | 决策 |
|------|------|
| 产品形式 | 网页应用（单 HTML 文件） |
| 原文来源 | 用户手动粘贴 |
| 反馈维度 | A. 识别错误（漏/多/错词）<br>B. 发音评分<br>C. 流畅度<br>D. 语调/重音 |
| 语音引擎 | Azure Speech Services（发音评估） |
| API Key 处理 | 用户在页面内填写自己的 Azure Key 和 Region（测试阶段最简方案） |
| 练习模式 | 混合模式：先全文朗读看问题，再针对问题句/词反复练习 |
| 主要使用设备 | iPhone Safari（需重点测试） |

## 3. 架构

单文件网页应用，无后端。

```
┌─────────────────────────────────────┐
│  朗读练习网页（HTML + CSS + JS）      │
│  - 配置区                            │
│  - 文本区                            │
│  - 录音控制                          │
│  - 结果高亮                          │
│  - Drill 面板                        │
└──────────────┬──────────────────────┘
               │ 浏览器内录音 + HTTPS
               ▼
┌─────────────────────────────────────┐
│  Azure Speech Services              │
│  - 语音识别                          │
│  - 发音评估（准确度/流畅度/语调）     │
└─────────────────────────────────────┘
```

## 4. 界面流程

1. **准备页面**：用户粘贴原文，填写 Azure Region 和 Subscription Key。
2. **录音页面**：显示原文，实时录音计时，可停止。
3. **结果页面**：展示总体分数（总分 + 准确度/流畅度/语调），原文按词高亮（绿色/黄色/红色）。
4. **Drill 页面**：点击问题句/词后进入，显示目标片段、标准发音按钮、录音按钮，达到目标分数后进入下一个。

## 5. 数据流

1. 用户填写配置和原文，点击“开始朗读评估”。
2. 浏览器请求麦克风权限并开始录音。
3. 音频流送入 Azure Speech JS SDK（发音评估模式）。
4. Azure 返回识别文本 + 词级/音素级评分。
5. 前端将原文与识别结果对齐，忽略大小写、标点、缩写（如 I’ve → ive）。
6. UI 按规则高亮（默认阈值，可调整）：
   - 绿色：单词发音分 ≥ 80
   - 黄色：60 ≤ 单词发音分 < 80，或识别出多余词
   - 红色：漏读、错读，或单词发音分 < 60
7. 用户点击问题句/词进入 Drill 模式，反复练习直到该片段平均分 ≥ 80 后解锁下一个。

## 6. 对齐规则

- 归一化：转小写、移除标点、统一缩写（I’ve → ive，I’m → im）。
- 句子拆分：按 `.` `!` `?` 分割，缩写末尾的小数点需简单保护（如 "U.S."）。
- 顺序匹配原文词与识别词。
- 漏词：原文有、识别无 → 红色。
- 错词：位置相同但文本不同 → 根据 Azure 发音分决定红/黄。
- 多词：识别有、原文无 → 黄色提示。

## 7. 错误处理

| 场景 | 用户提示 |
|------|----------|
| 麦克风权限被拒绝 | 请允许浏览器使用麦克风，或检查地址栏权限设置 |
| Azure Key/Region 错误 | Key 或 Region 无效，请检查 Azure 门户 |
| 网络断开或超时 | 连接 Azure 失败，请检查网络后重试 |
| 没有检测到语音 | 未听到声音，请靠近麦克风再试一次 |
| 识别结果与原文差异过大 | 建议先听标准发音，再一句一句练 |
| 浏览器不支持 | 请使用最新版 Chrome/Edge/Safari |

## 8. 测试策略

1. **静态冒烟测试**：HTML 可直接打开，CDN 可加载，手机布局正常。
2. **模拟数据测试**：用写死的 Azure 返回 JSON 验证 UI、对齐、高亮、Drill 切换。
3. **真实录音测试**：在 iPhone Safari 上测试麦克风权限、录音、Azure 返回、反复 Drill。

## 9. 非目标（本版本不做）

- 用户账号/登录
- 历史记录持久化
- 后端服务或 Key 代理
- 文件上传/URL 抓取
- 多语言支持（仅 en-US）

## 10. 后续可扩展

- 接入本地后端代理隐藏 Azure Key。
- 增加历史记录和进步曲线。
- 支持上传文件自动提取文本。
- 支持更多评估维度或自定义评分阈值。
